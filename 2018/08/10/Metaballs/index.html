Moving Circles

Circle movement around screen can be easily modeled with Lissajous curves.

Sounds fancy, but it is just sin function that takes some variable, in this case time.
Output of function will be x and y coordinates.

In shadertoy there is uniform variable for time iTime.

Just change c definition to

And that’s it. Main part is vec2(sin(iTime) * 0.5) Time is constantly increasing and as you already know sin is returning value between -1 and 1 for any input value. By multiplying it with 0.5, values -1 to 1 becaome -0.5 to 0.5, which is perfect for current view (since edges are going roughly from -0.5 to 0.5 … give or take a bit due to aspect ratio).
