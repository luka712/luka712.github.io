Intro

What is dither? In this context, it is a way of dealing with limited color pallete and at the same time creating illusion of color depth.

Take a look at image here:

You may notice that there are only 2 colors, red and blue. But at square get smaller, we percieve it as purple. That’s what dithering is trying to achieve. If you’re intrested in different ditering algorithms it is easily found on wikipedia. I’m going to go through ordered dithering.

This is effect that dither will have on image

What you might not notice, is that only 2 colors are used here, black and white. That’s how dither effect will work.
We’re going to turn image into grayscale and then compare grayscale to values in threshold matrix. If it passes them, white is returned, if not black will be returned.

Now what is Grayscale? You might think that’s it’s color, but it is actually amount of light. We just have white as strongest shade and black as weakest. More here.

We can get luminance with formula:

Y=0.2126R+0.7152G+0.0722B

where R,G and B are red, green and blue colors. You might also notice, that whole expression seems like dot product between two vectors.

Y = dot(col, lum)
Y = col.x lum.x + col.y lum.y + col.z * lum.z ( instead of xyz, rgb can be used).

As for dividing by 16, we’re doing that to put matrix values in 0 to 1 space ( techinally close to 1, since biggest value in matrix is 15). v < t is comparison between current value v and threshold value t. If v is less then t, return 0 for black, else return 1 for white.

Now we need to find x and y. X is simply index from 0 to 3 and same is true for y. This simply means that each 4x4 pixels chunk of screen is going to be compared to dither matrix. Final step is to compare luminance with matrix by calling findClosest function.

One thing that I failed to mention is that matrix values are entered in column order. That means that 4 numbers (0., 8., 2., 10.) from our first row are actually column. That can be easily changed in findClosest method by transpose operation.
